# 影视设备打分 - 产品需求文档 v2.2

## 1. 概述

### 1.1 产品定位
面向影视行业从业者和摄影爱好者的**设备组合推荐与评分系统**。用户在输入预算和使用需求后，系统自动推荐最优的机身+镜头组合方案，并给出多维度评分。

### 1.2 目标用户
- 准备购买相机系统的用户（不知道买什么）
- 已有机身想配镜头的用户
- 已有镜头想配机身的用户
- 想对比不同组合方案的客户

### 3.3 价格数据结构

```json
{
  "launchPrice": { "low": 15000, "high": 17000 },
  "usedPrice":  { "low": 8500,  "high": 11000 }
}
```

- `launchPrice`：发布时官方价格区间
- `usedPrice`：目前二手市场行情区间
- 两个价格字段均为 `{ low, high }` 对象
- 计算组合总价时默认取 `low` 值

## 4. 双标签在推荐中的使用

两套标签各司其职：

```
特性标签 (featureTags)    场景标签 (sceneTags)
    ↓                            ↓
"这个设备有什么能力"      "这个设备适合干什么"
    ↓                            ↓
  预算过滤、用户筛选        推荐匹配、评分加权
```

**用户端的体现：**

在需求输入和结果展示中，两套标签分别在两个地方发挥作用：

**输入阶段：**
- 用户选择拍摄类型 → 匹配设备的 sceneTags
- 用户选择画幅/品牌 → 过滤
- 用户看设备详情 → 展示 featureTags 让用户快速了解设备特性

**推荐阶段：**
- 匹配加分：组合的机身/镜头 sceneTags 与用户拍摄类型匹配 → 每个匹配 +3 分
- 展示标签：推荐结果卡片上显示关键 featureTags（如"高像素"、"大光圈"、"轻便"）
- 标签作为推荐理由的素材： "高像素机身搭配大光圈镜头，适合人像拍摄"

## 5. 标签分配规则（自动 + 人工）

### 5.1 机身标签自动分配逻辑

```javascript
function assignBodyTags(body) {
  const tags = { feature: [], scene: [] };

  // ---- 特性标签 ----
  if (body.megapixels >= 40) tags.feature.push("high-mp");
  else if (body.megapixels >= 25) tags.feature.push("mid-mp");
  else tags.feature.push("low-mp");

  if (body.fps >= 20) tags.feature.push("high-fps");
  else if (body.fps >= 10) tags.feature.push("mid-fps");

  if (body.video.includes("8K")) tags.feature.push("8k-video");
  else if (body.video.includes("6K")) tags.feature.push("6k-video");
  if (body.video.includes("120p")) tags.feature.push("4k120-video");
  if (body.videoRaw) tags.feature.push("raw-video");

  if (body.weight < 600) tags.feature.push("lightweight");
  if (body.weight > 1000) tags.feature.push("heavy");

  // 价格评估
  const launchLow = body.launchPrice?.low || 10000;
  if (launchLow >= 30000 || body.fps >= 20) tags.feature.push("pro-body");

  if (body.ibisLevel) {
    const stops = parseInt(body.ibisLevel);
    if (stops >= 7) tags.feature.push("pro-ibis");
  }
  if (body.sensor?.includes("堆栈式")) tags.feature.push("stacked-sensor");

  const isoUpper = parseInt(body.iso?.split("(")[1]?.replace(/[^0-9]/g,"") || "0");
  if (isoUpper >= 102400) tags.feature.push("high-iso");

  // ---- 场景标签 ----
  if (body.megapixels >= 40) {
    tags.scene.push("portrait", "landscape", "commercial");
  } else if (body.megapixels >= 24) {
    tags.scene.push("wedding", "documentary", "street");
  }
  if (body.video.includes("8K")) tags.scene.push("cinema", "documentary");
  else if (body.video.includes("6K")) tags.scene.push("cinema");
  if (body.video.includes("120p")) tags.scene.push("sports");
  if (body.weight < 600) tags.scene.push("vlog", "street", "landscape");
  if (body.fps >= 20) tags.scene.push("sports", "wildlife");

  return tags;
}
```

### 5.2 镜头标签自动分配逻辑

```javascript
function assignLensTags(lens) {
  const tags = { feature: [], scene: [] };

  // ---- 特性标签 ----
  const f = parseFloat(lens.maxAperture.replace("F", ""));
  if (f <= 1.4) tags.feature.push("large-aperture");
  else if (f <= 2) tags.feature.push("mid-aperture");
  if (lens.type === "变焦" && (f === 2.8 || f === 4)) tags.feature.push("constant-aperture");

  if (lens.weight < 400) tags.feature.push("lightweight");
  if (lens.weight > 800) tags.feature.push("heavy");

  if (lens.isL || f <= 1.4 || (lens.type === "变焦" && f === 2.8)) tags.feature.push("pro-lens");

  // 焦段判断
  const lower = parseFloat(lens.focalRange.split("-")[0]);
  const upper = parseFloat(lens.focalRange.split("-")[1] || lens.focalRange);
  // 简化判断：全画幅等效焦段
  if (lower <= 24 || (lens.type === "定焦" && lower <= 24)) tags.feature.push("wide-angle");
  if (upper >= 135 || (lens.type === "定焦" && lower >= 135)) tags.feature.push("telephoto");
  if (lens.focalRange.includes("50") || lens.focalRange.includes("85") || lens.focalRange.includes("135")) {
    tags.feature.push("standard");
    if (lens.focalRange.includes("85") || lens.focalRange.includes("135")) {
      tags.feature.push("portrait-focal");
    }
  }
  if (lens.isIS) tags.feature.push("is-lens");
  if (lens.type === "变焦" && lower <= 24 && upper >= 70) tags.feature.push("travel-zoom");
  if (lens.filterSize === 82) tags.feature.push("filter-82");

  // ---- 场景标签 ----
  if (f <= 1.4) tags.scene.push("portrait");
  if (lens.isIS) tags.scene.push("vlog", "documentary");
  if (lens.type === "变焦" && f <= 2.8) tags.scene.push("wedding", "documentary");
  if (lens.weight < 400) tags.scene.push("vlog", "street", "landscape");
  if (tags.feature.includes("telephoto")) tags.scene.push("sports", "wildlife");
  if (tags.feature.includes("wide-angle")) tags.scene.push("landscape", "street");
  if (lens.type === "定焦" && f <= 1.4) tags.scene.push("street");

  return tags;
}
```

### 5.3 人工标注覆盖

自动分配的标签可以手动覆写。在数据文件中直接指定 `featureTags` 和 `sceneTags` 数组时，以手动指定为准，跳过自动分配。

## 7. 推荐算法（完整版）

### 7.1 推荐总分公式

```
推荐总分 = 需求匹配分 × 系统质量分 / 100
```

### 7.2 需求匹配分

根据用户照片/视频比例确定基础权重：

| 用途类型 | 画质 | 视频 | 便携 | 性价比 | 专业度 |
|---------|------|------|------|--------|-------|
| 默认均衡 | 30% | 20% | 15% | 15% | 20% |
| 照片优先 (≥70%) | 40% | 5% | 15% | 20% | 20% |
| 视频优先 (≥70%) | 10% | 40% | 20% | 15% | 15% |
| 纯视频 (≥90%) | 5% | 50% | 20% | 10% | 15% |

拍摄类型调整因子叠加后归一化。

### 7.3 标签匹配加分

每个组合的 sceneTags 与用户选择的拍摄类型进行匹配：

- 每个匹配的标签：+3 分
- 上限：+15 分
- 加分计入需求匹配分

### 7.4 预算过滤

- 全新/二手模式决定用哪套价格
- 总价在预算内：正常展示
- 总价超出 ≤ 15%：标记"略超预算"
- 总价超出 > 15%：隐藏

### 7.5 推荐理由生成

模板 + 标签填充：

| 场景 | 模板 |
|------|------|
| 高预算专业 | "【机身】搭配【镜头】，【featureTags】标签组合，适合【sceneTags】拍摄" |
| 预算适中 | "性价比之选，【优势维度】得分突出，适合【拍摄类型】" |
| 低预算入门 | "入门优选，轻便便携，适合日常【拍摄类型】" |
| 特定场景推荐 | "【机身】的【bodyTag】x【镜头】的【lensTag】，【拍摄类型】利器" |
